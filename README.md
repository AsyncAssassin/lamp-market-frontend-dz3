# LampMarket. Домашнее задание 3

Пользовательская часть интернет-магазина лампочек на React.

Проект сделан для ДЗ3: страницы каталога, карточки товара, корзины, оформления и подтверждения заказа. Серверная часть на этом этапе не подключается, данные находятся в тестовых файлах.

## Технологии

- React
- Vite
- React Router DOM
- CSS
- `localStorage` для корзины

## Запуск

```bash
npm install
npm run dev
```

Локальный адрес Vite по умолчанию:

```text
http://localhost:5173
```

Если порт занят, Vite покажет в терминале другой свободный адрес.

Проверка сборки:

```bash
npm run lint
npm run build
npm run preview
```

## Страницы

| Маршрут | Страница |
|---|---|
| `/` | каталог |
| `/catalog` | каталог |
| `/products/:id` | карточка товара |
| `/cart` | корзина |
| `/checkout` | оформление заказа |
| `/order-success` | подтверждение заказа |
| `*` | страница 404 |

## Что реализовано

- 5 категорий и 20 тестовых товаров; в каталоге показываются активные товары.
- Фильтрация каталога по категориям.
- Поиск товаров по названию.
- Карточка товара с характеристиками.
- Добавление товара в корзину.
- Изменение количества и удаление позиции.
- Сохранение корзины после перезагрузки через `localStorage`.
- Оформление заказа без авторизации.
- Сохранение последнего тестового заказа в `sessionStorage`.
- Адаптивная верстка от 320px.

## Что не реализуется в ДЗ3

- Панель управления.
- Авторизация и регистрация.
- Личный кабинет и история заказов.
- Подключение серверной части.
- Оплата.

Эти части исключены по условию задания. Репозиторий содержит только клиентское приложение.

## Проверка перед сдачей

1. Открыть `/catalog`, проверить категории и товары.
2. Открыть карточку товара через каталог.
3. Добавить доступный товар в корзину из каталога и из карточки.
4. Изменить количество в корзине.
5. Удалить позицию из корзины.
6. Перейти к оформлению заказа.
7. Проверить ошибки пустой формы.
8. Заполнить форму и подтвердить заказ.
9. Проверить страницу `/order-success`.
10. Открыть `/products/999` и неизвестный маршрут, проверить состояния ошибок.
11. Проверить ширины 320px, 375px, 768px, 1024px и 1440px.
